## P1 — 高優先度

### OGPイメージ (X/Twitterカード)
**What:** 各ページに固有のOGP画像を設定。XでシェアしたときにURL文字列ではなく綺麗なカードが表示されるようにする。
**Why:** ブランディング目的のサイトなのに、Xシェア時に何も表示されない。倫 / Rinの世界観が一切伝わらない。
**Pros:** Xからの流入改善、シェアされたときのブランド訴求力アップ。
**Cons:** OGP画像のデザイン・生成が必要（next/og でコードから生成できる）。
**Context:** 現在 `app/[route]/layout.tsx` にメタデータは設定済み。OGP `images` フィールドを追加するだけ。next/og か静的画像どちらでも可。
**Effort:** S → CC: ~15分
**Depends on:** なし

## P2 — 中優先度

### YouTube埋め込みプレーヤー
**What:** 作品カードのサムネイルをクリックしたとき、YouTubeに遷移するのではなくサイト内でモーダル/インライン再生できるようにする。
**Why:** 音楽プロデューサーのサイトなのに音が聴けない。ユーザーがサイトを離れて戻ってこない。
**Pros:** 「透明感のある音づくり」をサイト上で体験させられる。離脱率低下。
**Cons:** YouTube IFrame API の実装が必要。モーダルのUX設計が必要。
**Context:** `app/top/page.tsx` の `works` 配列と `app/works/page.tsx` に対象データあり。サムネイルURLはすでに `i.ytimg.com` 経由で表示中。
**Effort:** M → CC: ~20分
**Depends on:** なし

### 共通コンポーネントDRY化
**What:** `GlassOrb`, `AnimatedPanel`, `PageFrame` (背景グラデーション+orbs), `ContactBlock`, `SectionHeader` を `components/site/` に共通化。
**Why:** 現在5〜7ファイルに同一実装がコピーされている。デザイン変更のたびに全ファイルを個別編集する必要がある。
**Pros:** 保守性が大幅向上。将来のデザイン変更が1箇所で済む。
**Cons:** 全ページファイルに影響するため慎重なリファクタリングが必要。
**Context:** 対象: `app/top`, `app/mix`, `app/guide`, `app/audio-check`, `app/works`, `app/instrumentals`, `app/compose`。AnimatedPanel は mix/guide/audio-check で型定義も重複。
**Effort:** M → CC: ~20分
**Depends on:** なし

### 料金シミュレーター
**What:** MIX依頼ページに、簡単な選択肢（ボーカルMIXのみ/ハモリあり/追加対応など）で概算金額が出るインタラクティブな見積もりUIを追加。
**Why:** 「費用感がわからないと相談しづらい」という初回ユーザーの心理的障壁を下げる。

**Cons:** 価格が変わったときに更新が必要。ユーザーが「この価格で固定」と思い込むリスク。
**Context:** `app/mix/page.tsx` の `pricing` 配列 (line 23-27) を元にUIを構築できる。`lib/contactPolicy.ts` にも連絡方法の記述あり。

### 音声チェック Web Worker 最適化
**What:** `lib/audioAnalysis.ts` の分析処理（ノイズフロア計算、ピーク検出）をWeb Workerに移行。
**Why:** 大きなファイル（100MB）でメインスレッドが詰まるのを防ぐ。現在は `AudioContext` がメインスレッド必須なため解析のみWorkerへ。
**Context:** `Float32Array` は transferable。`decodeAudioData()` はメインスレッドのまま、decoded後のデータをWorkerにpostMessageで転送する。
**Effort:** M → CC: ~20分
**Depends on:** 音声チェックツール実装済み (done: 2026-04-06)
